## Kernel SU Rooting
오늘은 S20 기기를 루팅해보려고 한다. 먼저 Kernel SU가 무엇인지부터 알아보자.

### Kernel SU ?
Kernel SU(Kernel Assisted Superuser)는 루트 권한을 얻기 위한 루팅 방식으로 루트 권한 관리 로직이 유저스페이스가 아닌 <u>커널(Kernel)에 직접 구현되어</u> 다른 루팅 방식(ex. Magisk)에 비해 탐지가 쉽지 않다.
또한 Kernel SU는 \*Kprobes기반 시스템 호출 후킹을 통해 루트 권한을 제공하며 오버레이 마운트(\*overlayfs) 방식으로 처리되어 실제 시스템 파티션을 수정하지 않는 시스템 리스 구조를 유지한다.

* Kprobes: 리눅스 커널에서 제공하는 동적 디버깅 기능, 커널 함수를 후킹해 로직 삽입 가능
* overlayfs: 원본 파티션은 읽기 전용으로 유지하며 그 위에 쓰기 가능한 레이어를 덮어 시스템을 수정한 것처럼 보이게 하는 파일 시스템 중첩 기능

## Rooting Start(feat. KernelSU)
우선 설정 > 휴대전화 정보 > 소프트웨어 정보 > 빌드번호를 연타하여 개발자 옵션을 활성화해준다. <br>
활성화 된 개발자 옵션에 들어가면 <u>OEM 잠금 해제라는 기능</u>이 있다. 해당 기능을 활성화해준다.

\+ 그 아래 USB 디버깅도 같이 활성화

<br>

Github에 올라와있는 설치 방법은 엄청 간단하다.
```
1. 커널 파일과 vbmeta_disabled.tar 파일을 다운로드하세요.
2. 휴대폰에서 다운로드 모드로 진입하세요 (adb reboot download 명령어를 입력하거나 단축키를 누르세요).
3. Odin을 실행하세요 (3.14.1 버전을 권장합니다).
4. AP 슬롯 -> 커널 파일 // USERDATA 슬롯 -> vbmeta_disabled.tar 파일
5. 시작 버튼을 클릭하세요
6. 부팅이 성공할 때까지 기다리세요. (부팅 루프가 발생하면 복구 모드로 진입하여 데이터 초기화(공장 초기화)를 한 후 시스템을 재부팅하세요.)
7. KernelSU-Next Manager 앱 설치(최신 버전 권장) -> SuSFS-webui 모듈 설치
8. 재부팅! 축하합니다! 당신의 휴대폰에 KernelSU-Next + SUSFS가 설치되었습니다.
```

S20이므로 **x1q_SuSFS.tar**과 **vbmeta_disabled.tar** 파일을 다운로드한다. <br>
(vbmeta_disabled.tar: Verified Boot 검증을 비활성화해 커스텀 커널이나 루팅 적용 시 부팅)

<br>

Odin으로 밀어넣기 전 <span style="color: red; font-weight: bold">📍 부트로더 언락 작업</span>이 필요하다. <br>
```Unlock
1. 볼륨 상 + 볼륨 하 + 전원 키를 몇 초간 꾹 PUSH
2. 경고화면이 나오면 볼륨 상 키로 계속
3. 이후 Unlock bootloader? 라는 화면이 나오면 마찬가지로 볼륨 상 키(yes)로 부트로더 언락!
```

<br>

이후 볼륨 상 + 볼륨 하 + 전원 키를 누르면 다운로드 모드로 진입한다. 이때 Odin으로 밀어넣어줘야한다.
![Odin](/assets/images/source/260216_img/image.png "Odin")
